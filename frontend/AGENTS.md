# Frontend — Agent Rules

See root [AGENTS.md](../AGENTS.md) for project-wide context, PM working-agreement, safety/tone rules, and the Orchestration Map. This file is **frontend-specific** only.

## What lives here

Next.js 16 (App Router) consumer-facing web app. Connects to the Python ADK backend over HTTP. Deployed as its own Cloud Run service.

## Source of truth for design + copy

**[docs/design-brief.md](../docs/design-brief.md) is non-negotiable.** Read it before opening a component file, writing a string of copy, or picking a color. Specifically:
- Archetype: premium consumer health (Headspace · Maven Clinic · One Medical · Calm · Apple Health). NOT classical insurance portals, enterprise health software, chatbot UIs, generic SaaS dashboards.
- Voice rules in design-brief §4 are hard rules — no "AI assistant" framing, no exclamation marks, no chatbot enthusiasm, no insurance jargon without translation, "person" not "human", no manufactured urgency.
- Anti-pattern list in design-brief §8 is a veto list — if you see one of those in a PR, block.

## Design system handoff

Generated by the `frontend-design` skill chain (Session 7). The contract files live in `.design/aegis/` at the repo root; the application copies of the live tokens are in `src/styles/`:

- [`.design/aegis/ARCHETYPE.md`](../.design/aegis/ARCHETYPE.md) — design philosophy + reference sites
- [`.design/aegis/TOKENS.md`](../.design/aegis/TOKENS.md) — token rationale + banned-defaults audit
- [`.design/aegis/ICONS.md`](../.design/aegis/ICONS.md) — icon strategy (tuned Lucide + 8 bespoke)
- [`src/styles/tokens.css`](src/styles/tokens.css) — runtime CSS custom properties (light + dark)
- [`src/styles/tokens.ts`](src/styles/tokens.ts) — typed motion + spatial constants

The `.design/aegis/` files are the spec; `src/styles/` is the consumed copy. When tokens change, edit `.design/aegis/` first, then sync.

## Stack

| Layer | Tool | Note |
|---|---|---|
| Framework | Next.js 16 (App Router) | SSR for landing page SEO |
| Styling | Tailwind v4 + design tokens (CSS variables via `@theme`) | Tailwind defaults (`slate`, `zinc`, `gray`, etc.) are forbidden in app code |
| Components | shadcn/ui | **Copied into repo and customized, NOT installed as a dependency.** We own the components. |
| Motion | framer-motion | Long durations, soft easing (`cubic-bezier(0.2, 0.8, 0.2, 1)`, 400ms default) |
| Icons | Lucide React (tuned subset, see `src/icons/lucide.tsx`) + ~6–10 bespoke SVGs | Avoid the "Lucide everywhere" vibecoded look |
| Fonts | self-hosted via `next/font` (Source Serif 4 + Inter + JetBrains Mono) | No Google Fonts CDN flash |
| Deployment | Google Cloud Run | Min instances = 1 during demo period |

## Non-obvious patterns

- **shadcn is copied, not imported.** When you add a shadcn component, copy the source into `src/components/ui/` and customize against the tokens. Never `pnpm add @shadcn/ui` (that package shouldn't even exist as a runtime dep).
- **Design tokens are CSS variables.** All colors, type scale, spacing, radius, motion curves live as CSS custom properties in `src/styles/tokens.css`. Never hardcode hex values or use Tailwind palette names like `bg-slate-900`.
- **Lucide imports must go through `src/icons/lucide.tsx`.** This wrapper applies the project stroke / size / color rules. Importing directly from `lucide-react` in a component is a lint error.
- **Motion respects `prefers-reduced-motion: reduce`.** The token CSS already collapses durations to 1ms when reduced motion is requested; framer-motion `useReducedMotion()` covers the spring/keyframe cases.
- **No tracking / analytics** beyond the bare minimum needed for the product to function. Consider whether you need GA at all — probably not.
- **No Gemini / Phoenix / ADK branding in user-facing UI.** Those belong in README and Devpost, not the product surface.

## Code style

- TypeScript strict mode.
- Server Components by default; mark Client Components explicitly with `'use client'`.
- Co-locate component + styles; one component per file.
- Accessibility floor: WCAG 2.2 AA contrast, full keyboard nav, visible focus states, `<label>` on every input, semantic landmarks.

## Boundaries

- **Ask first:** new dependencies, design system changes, new pages, new color/typography tokens, copy on user-facing surfaces.
- **Full autonomy:** test writing, internal refactors, lint/format fixes, behaviour-preserving cleanups.
- **Never:** AI-marketing copy, urgency-manipulation patterns, dark patterns, stock photos of multi-ethnic smiling people in hospital corridors, the phrase "your insurance journey".
